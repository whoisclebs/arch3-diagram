# Regras de lint v0.1

As regras de lint do Arch3 são convenções consultivas acima da validação de
schema e semântica.

## Regras atuais

- ids de containers devem usar kebab-case
- ids de components devem usar kebab-case
- containers devem declarar `metadata.owner`
- containers devem declarar `metadata.tier`
- containers devem declarar pelo menos um relacionamento
- components devem declarar pelo menos uma library
- components devem declarar `metadata.owner`
