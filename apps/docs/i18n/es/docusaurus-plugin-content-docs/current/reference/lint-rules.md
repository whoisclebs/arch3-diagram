# Reglas de lint v0.1

Las reglas de lint de Arch3 son convenciones asesoras por encima de la
validación de schema y semántica.

## Reglas actuales

- ids de containers deben usar kebab-case
- ids de components deben usar kebab-case
- containers deben declarar `metadata.owner`
- containers deben declarar `metadata.tier`
- containers deben declarar al menos una relación
- components deben declarar al menos una library
- components deben declarar `metadata.owner`
